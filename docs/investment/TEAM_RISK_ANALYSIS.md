## 三、招聘风险 🟡 中 (5%)

### 3.1 招聘挑战

#### 难点分析

```
1. 资金限制
   - 无法支付市场薪资
   - 只能给股权
   - 吸引力有限

2. 品牌知名度
   - 新项目，无知名度
   - 难以吸引顶尖人才
   - 需要靠创始人个人魅力

3. 地理位置
   - 如果在二三线城市
   - 人才池有限
   - 需要远程/搬迁

4. 时间压力
   - 需要快速招人
   - 但招聘需要时间
   - 可能降低标准
```

**目标**: 招聘 1-2 人

**优先级 P0: 技术合伙人/CTO**
```
要求:
- 5+ 年芯片设计/FPGA 经验
- Chisel/Scala 熟练
- 认同项目愿景
- 愿意拿低薪 + 股权

薪资:
- 现金: $3K-5K/月
- 股权: 10-15%
- Vesting: 4 年

招聘渠道:
- GitHub (开源贡献者)
- Chisel 社区
- 大学校友
- 技术会议

时间: 1-2 个月
```

## 四、团队协作风险 🟡 中 (5%)

### 4.1 从 1 人到多人的挑战

#### 文化冲突
```

- 新人适应困难
- 沟通成本增加
- 决策效率下降

- 明确沟通机制
- 建立决策流程
- 定期团队建设
```

```
风险:
- 创始人不愿放权
- 新人无法发挥
- 职责不清晰
- 互相推诿

缓解:
- 明确岗位职责
- 授权清单
- OKR 管理
- 定期 1on1
```

#### 技术债务
```
风险:
- 代码只有创始人懂
- 文档不完善
- 新人上手困难
- 知识转移成本高

缓解:
- 完善代码注释
- 编写技术文档
- Pair Programming
- Code Review
```
